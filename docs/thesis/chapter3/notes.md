## Зафиксированные решения

- Проект описывается как реализованный прототип.
- Базовый классификатор: ML/ONNX.
- Гибридный подход: ML/ONNX + LLM fallback.
- Рекомендации генерируются Financial Coach Agent на основе агрегированных данных и LLM.
- Тексты пишутся по подразделам в отдельных Markdown-файлах.
- LLM-провайдер в описании прототипа: DeepSeek.
- Целевой пользователь в тексте главы: клиент банка.
- В финальную версию главы 3 входят четыре диаграммы: контекстная, логическая, последовательности и state.
- AI-логи хранятся только в файловой системе.
- В PostgreSQL через JSONB хранится только структурированный ответ LLM в контуре рекомендаций.
- Transaction Classifier Agent сохраняет Kafka-контракт снаружи, но для сложных случаев может выполнять retrieval похожих кейсов из `pgvector`.
- Для сложных ситуаций агент может выполнять retrieval похожих кейсов из `pgvector`, не меняя внешний Kafka-контракт.
