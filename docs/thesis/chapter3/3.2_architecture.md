# 3.2. Проектирование архитектуры прототипа

## Назначение подраздела

Подраздел описывает архитектуру прототипа FinSense, основные компоненты, их ответственность и способы взаимодействия.

## Черновая структура

### 3.2.1. Архитектурный подход

- Event-driven microservices architecture.
- Разделение синхронного и асинхронного контуров.
- ML-first стратегия с LLM fallback.
- Stateless-сервисы и обмен через Kafka/REST.

### 3.2.2. Состав компонентов

- Core Service.
- Classifier Service.
- Transaction Classifier Agent.
- Внутренний retrieval-контур агента через `pgvector` для сложных случаев.
- Financial Coach Agent.
- Notify Service.
- Kafka.
- PostgreSQL.
- Вспомогательные инструменты: Transaction Tester, Prompt Lab, ML Training.

### 3.2.3. Поток классификации транзакций

- Поступление события `raw-transactions`.
- Синхронный вызов Classifier Service.
- Проверка `confidence`.
- Публикация запроса в `llm-classifier-requests` при низкой уверенности.
- При необходимости поиск похожих сложных кейсов в `pgvector`.
- Получение ответа из `llm-classifier-responses`.
- Обновление состояния транзакции.

### 3.2.4. Поток генерации рекомендаций

- Создание запроса пользователем.
- Создание записи со статусом `PENDING`.
- Публикация `coach-requests`.
- Анализ данных Financial Coach Agent.
- Сохранение результата и публикация `coach-responses`.
- Доставка уведомления.

### 3.2.5. Хранение данных

- Схема `core`: пользователи, счета, транзакции.
- Схема `recommendations`: запросы и результаты рекомендаций.
- JSONB для структурированного ответа LLM в рекомендациях.
- AI-логи хранятся в файловой системе, а не в PostgreSQL.

### 3.2.6. Надёжность и трассируемость

- Correlation identifiers: `transactionId`, `requestId`.
- At-least-once доставка Kafka.
- Идемпотентность обработчиков.
- Статусы обработки транзакций и рекомендаций.

### 3.2.7. Архитектурные компромиссы

- Общая PostgreSQL база с разными схемами для прототипа.
- REST для быстрого ML-инференса.
- Kafka для LLM и асинхронных операций.
- Внешний LLM API вместо локального inference.
