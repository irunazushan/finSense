# 3.6. Результаты эксперимента и анализ эффективности внедрения AI-агентов

## Назначение подраздела

Подраздел будет заполнен после проведения экспериментов. Сейчас файл содержит структуру будущего анализа.

## Черновая структура

### 3.6.1. Результаты классификации транзакций

- Таблица качества базового ML/ONNX-классификатора.
- Таблица качества гибридного подхода.
- Сравнение по accuracy, macro F1 и weighted F1.
- Анализ ошибок по категориям.

### 3.6.2. Анализ fallback на LLM

- Доля транзакций, направленных в Transaction Classifier Agent.
- Доля случаев, в которых агент дополнительно обращался к `pgvector`.
- Типовые случаи, где LLM улучшает классификацию.
- Ошибочные или спорные случаи.
- Влияние порога confidence.

### 3.6.3. Производительность и задержки

- Latency ML-инференса.
- Latency LLM-классификации.
- End-to-end latency.
- p95 времени обработки.

### 3.6.4. Результаты генерации рекомендаций

- Примеры рекомендаций.
- Оценка полноты и полезности советов.
- Анализ влияния tools на контролируемость ответа.

### 3.6.5. Выводы по эффективности прототипа

- Сравнение базового и гибридного подхода.
- Оценка достигнутых преимуществ.
- Ограничения прототипа.
- Направления дальнейшего развития.

## Данные для заполнения

- Результаты ML evaluation.
- Логи Transaction Classifier Agent.
- Логи Financial Coach Agent.
- Выгрузки из PostgreSQL.
- Kafka statistics / счетчики событий.
